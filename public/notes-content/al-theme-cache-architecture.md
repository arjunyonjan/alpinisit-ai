# AI Theme Cache Architecture

This document describes the shared, file-based caching system for AI-generated HTML themes in our Next.js application.

## Architecture Diagram

![AI Theme Cache Architecture Diagram](https://res.cloudinary.com/dpnxmo8ak/image/upload/v1780550813/ChatGPT_Image_Jun_4_2026_11_11_29_AM_gvsn3x.png)

## How It Works

The system uses a "generate-once, serve-many" model:

1.  **Cache Miss (New Generation):**
    *   When the first user requests a theme for a note, the system calls the **DeepSeek AI API** (`POST /api/ai/enhance`).
    *   The user selects a style (Fun, Poetic, or Professional).
    *   The generated HTML (15-30 KB) is saved to the server's **File System Cache** (`public/themes-cache/`).

2.  **Cache Hit (Instant Serve):**
    *   When any subsequent user views the same note, the system serves the pre-generated HTML directly from disk via `GET /api/ai/theme-cache`.
    *   This provides an instant response with **no additional API calls**.

## Core Components

*   **Next.js App Router:** The application framework (`app/notes/read/page.tsx`).
*   **DeepSeek AI API:** Generates the HTML theme (cache miss path).
*   **File System Cache:** Stores generated themes on disk for future requests.
*   **API Routes:**
    *   `POST /api/ai/enhance`: For generating a new theme.
    *   `GET /api/ai/theme-cache`: For retrieving a cached theme.

## Cached Files

The cache directory (`public/themes-cache/`) stores files using the naming convention `{note-title}_{style}.html`.

*   `my-note_fun.html`
*   `my-note_poetic.html`
*   `my-note_professional.html`

## System Benefits

*   **Shared Cache:** A theme generated by one user is instantly available to all other users.
*   **Cost Efficient:** Eliminates duplicate API calls for popular content.
*   **High Performance:** Serves cached themes with sub-10ms latency from disk.
*   **Resilient:** Cached themes remain available even if the AI API is offline.
